# 流序 UI 精修进度

Updated: 2026-05-23

## 目标

在不增加使用复杂度的前提下，让流序保持冷静、清爽、有层次的视觉体验。每一次改动都优先服务于“用户一眼知道页面能做什么”，其次再增加数据、动效和细节质感。

## 阶段一：核心页面统一

- [x] 历史页增加专注概览、近 7 天趋势和月度归档。
- [x] 历史页文案改为面向用户的节奏提示，减少开发口吻。
- [x] 根据体验反馈移除收集页、任务页新增概览卡，保持原流程轻量。
- [x] 历史页样式压平，和收集/任务的列表风格保持一致。
- [x] 历史页趋势图改为展示每日真实专注时长，0 数据只显示基线。
- [x] README 改为上线前可读的产品说明。

## 阶段二：卡片质感继续精修

- [ ] 统一收集、任务、历史三类卡片的圆角、内边距、分割方式和弱背景。
- [ ] 优化空状态和筛选状态的提示文案。
- [ ] 减少过重边框，改用轻分层和柔和面底区分信息。
- [ ] 检查深色主题下文字、卡片、按钮的对比度。

## 阶段三：交互微动效

- [ ] 为关键按钮补充轻微按压反馈。
- [ ] 统一页面切换、筛选展开、卡片点击的动效节奏。
- [ ] 避免新增页面堆叠，保持主流程层级清晰。

## 阶段四：上线前体验收口

- [ ] 清理所有测试文案、临时数据和开发提示。
- [ ] 检查用户手册与应用实际流程一致。
- [ ] 真机走查主要路径：收集、整理、专注、完成、归档、复盘。
- [ ] 根据真机截图进行最后一轮排版微调。
