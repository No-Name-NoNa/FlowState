# Flow State 用户手册

Updated: 2026-05-16

Flow State 是一款围绕“今天怎么过”设计的计划类应用。它不是把所有事情堆成清单，而是帮助用户先收集、再安排到时间线上、然后进入专注，最后用复盘修正下一次计划。

当前版本定位为体验版：核心的收集、安排、专注、复盘、本地数据和基础设置流程已经可以串起来；真实系统提醒、发布签名、设备运行细节仍需要后续在 HarmonyOS 真机或模拟器上验证。

## 1. 核心思路

Flow State 的使用顺序建议保持简单：

1. 想到事情先放进 `收集`。
2. 每天打开应用先看 `今天`。
3. 需要整理时进入 `规划`，把收集箱或未来事项安排到今天。
4. 到时间后进入 `专注`。
5. 一天结束后看 `复盘`，判断计划是否合理。

应用的底部主导航以 `今天` 为第一入口。大多数情况下，用户不需要先理解完整功能，只要知道“先收集，后安排，再执行”即可。

## 2. 页面说明

### 今天

`今天` 是主页面，用来回答三个问题：

- 现在最应该做什么。
- 今天已经安排了多少时间。
- 是否还有未整理的收集箱任务或明天准备事项。

主要区域包括：

- 下一步行动卡：根据当前状态显示开始专注、去收集、去规划或去复盘。
- 今日负荷：显示已安排分钟数和每日容量关系。
- 时间线：按时间顺序显示今天的计划块。
- 常用模板：当今天为空时，可快速加入晨间规划、深度工作、收尾复盘等固定时间块。
- 明天预备：提示已经安排到明天的事项。

建议用法：每天打开应用先看 `今天`，如果没有任务，就从收集箱安排一个最小可执行任务。

### 收集

`收集` 用来快速记下暂时不想安排时间的事情。它只显示尚未排期的任务。

可做的事情：

- 新建任务。
- 设置优先级。
- 快速安排到今天或明天。
- 进入整理面板，补充备注、子任务、日期、时间和时长。
- 删除不再需要的收集任务。

注意：当任务被安排到今天、明天或其它日期后，它会从收集箱消失。这不是丢失，而是进入了 `今天` 或 `未来` 的时间安排。

### 规划

`规划` 用来把零散任务变成可以执行的今天计划。它适合早上开始工作前，或一天中需要重新调整节奏时使用。

可做的事情：

- 查看可安排来源：收集箱任务、未来事项、今天已安排时间。
- 自动生成计划建议。
- 选择、排除、上移、下移、调整任务时长。
- 将未来事项移动到今天。
- 确认安排后直接开始第一段专注，或回到今天查看完整时间线。

建议用法：不要一次把所有任务都塞进今天。先安排真正要做的几段，把不确定的留在未来或收集箱。

### 未来

`未来` 管理已经安排到今天之后的事项。

分组包括：

- 明天。
- 本周稍后。
- 更晚。

可做的事情：

- 查看未来计划。
- 编辑日期、时间、时长、备注和子任务。
- 移动到今天。
- 退回收集箱。

建议用法：不确定具体执行时间的任务先放到明天或未来，第二天再通过 `规划` 拉回今天。

### 专注

`专注` 是执行页面。它从今天的时间块进入，也可以在规划完成后自动进入。

可做的事情：

- 查看当前专注任务。
- 启动计时。
- 完成后记录实际专注分钟数。
- 写一句简短复盘。
- 如果后面还有任务，可直接开始下一段。
- 返回今天查看剩余安排。

建议用法：专注完成页只写真实有用的一句话，例如“低估了资料整理时间”或“下一次先关掉消息提醒”。

### 复盘

`复盘` 用来判断计划质量，而不是单纯展示数字。

当前支持：

- 今天范围。
- 本周范围。
- 完成、错过、结转、计划调整等数据摘要。
- 轻量节奏图。
- 空状态引导：没有可复盘数据时，引导用户先收集或规划。

建议用法：一天结束后只看关键洞察，不要把复盘变成新的负担。重点是发现估时是否偏差、计划是否过载、是否有太多任务反复结转。

### 设置

`设置` 用来调整体验偏好，并查看当前版本状态。

当前包含：

- 默认专注时长。
- 每日容量。
- 主题偏好。
- 提醒偏好存储。
- 本地数据状态。
- 体验版状态说明。
- 构建、设备 QA、系统提醒、签名状态说明。

注意：当前版本会保存提醒偏好，但不声称已经接入真实系统提醒。真实通知、闹钟、后台能力需要基于 HarmonyOS 官方 API 继续实现并在设备上验证。

## 3. 常见工作流

### 快速记录一个任务

1. 进入 `收集`。
2. 输入任务标题。
3. 选择优先级。
4. 保存。

如果暂时不知道什么时候做，停在这里即可。

### 把任务安排到今天

1. 在 `收集` 找到任务。
2. 点 `今天`，快速放入今天时间线。
3. 或点 `整理`，选择具体日期、开始时间和预计时长。
4. 回到 `今天` 查看时间线。

### 把任务安排到明天或更晚

1. 在 `收集` 点 `明天`，快速放入明天。
2. 或点 `整理`，选择其它日期。
3. 进入 `未来` 查看排期。

### 整理复杂任务

1. 在任务上点 `整理` 或 `编辑`。
2. 保留标题为用户一眼能识别的动作。
3. 展开任务细节。
4. 添加备注。
5. 拆成几个子任务。
6. 设置时间和预计时长。

建议标题写动作，例如“整理周报初稿”，不要写过大的目标，例如“工作”。

### 开始今天第一段专注

1. 进入 `今天`。
2. 查看下一步行动卡。
3. 点 `开始`。
4. 在专注页启动计时。
5. 完成后确认结果。

如果今天没有安排，先去 `收集` 或 `规划`，不要直接进入空专注。

### 从规划进入专注

1. 进入 `规划`。
2. 选择需要安排的任务。
3. 调整顺序和时长。
4. 确认安排。
5. 点 `开始第一段`。

这个流程适合早上集中把今天的几个关键时间块排好。

### 处理错过的时间块

1. 在 `今天` 查看错过或未完成的块。
2. 选择结转到明天，或重新编辑时间。
3. 当天结束后去 `复盘` 查看是否存在计划过载。

如果连续多天结转同类任务，说明任务可能太大、优先级不真实，或安排时间过短。

## 4. 任务状态说明

Flow State 中任务不是只有“在列表里”一种状态：

- 收集箱任务：还没有安排时间。
- 今天任务：已经安排到今天时间线。
- 未来任务：已经安排到明天、本周稍后或更晚。
- 专注完成：已经有执行记录。
- 错过或结转：计划发生偏差，需要重新安排或复盘。

因此，任务从 `收集` 消失通常代表它已经进入了有时间的位置。用户应该去 `今天` 或 `未来` 查找。

## 5. 本地数据和隐私

当前版本使用本地数据保存任务、排期、专注记录、归档和偏好。设置页会展示本地数据状态，便于判断当前体验数据是否存在。

当前未完成的方向：

- 数据导入导出。
- 多设备同步。
- 云端账户。
- 系统日历同步。

这些能力涉及权限、存储、网络或系统能力，后续应在官方文档和设备验证后再进入正式实现。

## 6. 当前限制

当前体验版不承诺以下能力已经完成：

- 真实系统提醒推送。
- 系统日历集成。
- 后台持续专注状态。
- 桌面小组件。
- 项目、标签、自然语言解析等进阶组织能力。
- 发布签名和商店级发布配置。
- 真机或模拟器完整 QA。

如果用户需要确认某项能力是否已完成，可优先查看 `设置` 的体验状态，或查看 `docs/MASTER_FEATURE_CHECKLIST.md`。

## 7. 推荐使用方式

- 每天只维护一个真正可执行的今天计划。
- 任务先收集，不急着安排。
- 规划时控制容量，不要把一天填满。
- 专注完成后只写一句高价值复盘。
- 一天结束后看复盘，不追求复杂统计。
- 不确定什么时候做的事情放未来，不要挤进今天。

Flow State 的目标不是让用户管理更多信息，而是让用户更快知道下一步该做什么。
