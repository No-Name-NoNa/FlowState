# Flow State 使用指南

Updated: 2026-05-16

完整用户手册见 `docs/USER_MANUAL.md`。本文件保留为快速上手指南。

Flow State 是一个时间线优先的计划应用。它不是把任务堆成清单，而是帮助你把要做的事排进今天的时间线，然后一段一段执行。

## 入口

打开应用先看底部第一个页面：`今天`。

这里回答三个问题：

- 现在该做什么？
- 今天已经安排了多少？
- 有没有还没整理的任务？

## 最短流程

1. 进入 `收集`，写下一个任务。
2. 点任务上的 `今天`、`明天`，或点 `整理` 选择具体日期、时间和时长。
3. 回到 `今天`，查看时间线。
4. 到点后点 `开始`，进入专注页面。
5. 专注完成后先看计划/实际/偏差，可以写一句复盘；如果今天还有下一段，可以直接点 `开始下一段`。
6. 以后要做的事去 `未来` 查看，做完后在 `复盘` 看完成情况和计划变化。

如果今天还没有固定安排，`今天` 页面会出现 `常用模板`。点 `加入` 可以快速创建晨间规划、深度工作或收尾复盘这类固定时间块；同一个模板今天加入后不会重复显示。

## 为什么任务会从收集箱消失

收集箱只显示“还没安排时间”的任务。

当你点 `今天` 或 `明天` 后，任务会被转成一个时间块，所以会从收集箱移走：

- 点 `今天`：进入今天的时间线。
- 点 `明天`：进入 `未来` 的 `明天` 分组，也会在 `今天` 页面下方的 `明天预备` 区域提示。
- 点 `整理`：可以改标题、优先级、备注、子任务，也可以手动选择日期、结束时间和预计时长。

任务被安排以后，备注和子任务不会丢。`今天` 的时间块和 `未来` 的计划项会继续显示备注摘要和子任务进度；能看到子任务芯片时，可以直接点它完成或取消完成。

如果排期后还想补充任务内容，点时间块或未来计划项里的 `编辑`。默认先显示标题、日期、时间和时长；需要补充备注、优先级、子任务时，再展开 `任务细节`。

## 页面职责

- `今天`：主页面，用来查看下一段、今日负荷、时间线、明日预备和收集箱入口。
- `收集`：快速记录暂时不想安排的任务。
- `规划`：把收集箱任务和已经排到未来的事项重新整理进今天，适合早上或重新调整节奏时使用。
- `未来`：查看明天、本周稍后和以后已经安排的任务，也可以编辑时间、移到今天或退回收集箱。
- `复盘`：看今天做了什么、错过了什么、计划怎么变化，以及下一次该怎么调整。
- `设置`：调整默认专注时长、每日容量、主题和提醒偏好，也可以查看当前体验状态和本地数据状态。

## 当前建议用法

- 想到任何事先放进 `收集`，不要马上打断当前时间线。
- 如果一个任务比较大，先点 `整理` 拆成几个子任务，再安排进时间线。
- 任务排进 `今天` 或 `未来` 后，继续在时间块上查看备注和子任务进度；只想勾掉一个小步骤时，不必回到收集箱。
- 排期后的任务仍然可以在 `编辑` 面板里展开 `任务细节`，补备注、改优先级、拆子任务，不需要退回收集箱再处理。
- 每天开始先看 `今天`，如果空，就从 `收集` 安排一个任务。
- 如果每天都要做类似的固定事项，先用 `常用模板` 加入今天，再补充其它临时任务。
- 任务多的时候进 `规划`，先看 `可安排来源`：这里会同时显示收集箱数量、未来事项数量和今天已经安排的分钟数。
- 如果某个未来事项今天就该处理，可以在 `规划` 里点 `移到今天`，它会移动原来的安排，不会重复创建任务。
- 如果收集箱是空的，但未来已经有安排，`规划` 会优先建议把未来事项提前到今天，而不是让你重新创建任务。
- 用自动编排生成几个时间块；只勾选今天真的要做的，也可以上移、下移、加减时长。不合适的点 `排除`，任务不会删除，只会留在收集箱。
- 在 `规划` 里确认安排后，会出现第一段任务。点 `开始第一段` 会直接进入专注并开始计时；点 `回到 Today` 可以先看完整时间线。
- 在专注完成页里写复盘不会继续增加实际专注时长；实际分钟数会以点完成的时刻为准。
- 不确定什么时候做的任务先放 `明天`，再在第二天早上整理。
- 一天结束后看 `复盘`，重点看 `关键洞察`：它会提示是否超载、是否有冲突、结转是否太多、模板是否真的帮你减少了重复安排。
- 如果你在体验阶段不确定某项能力是否已完成，去 `设置` 看 `体验状态`。这里会明确区分已可体验的主流程、等待真机验证的部分，以及尚未接入的真实系统提醒。
