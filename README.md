
# 核心设计理念
摒弃传统臃肿的 GTD 项目子任务嵌套，采用完全扁平化、以执行为导向的节点化模型，将任务流转与专注计时深度融合，并赋予用户极高的数据控制权。

一、 全局通用基础特性 (系统级底层支持)
1. 泛在的检索与过滤系统
    - 几乎所有核心模块（任务流、收集箱、历史归档等）均配备了搜索栏与 `#标签` 筛选器 (Chips)。
    - 无论处于应用何处，均支持通过关键字或标签组合进行快速检索，确保在海量数据下实现秒级精准定位。

二、 核心任务流管理 (任务页)
1. 智能排序与列表展现
    - 抛弃重度卡片 UI，采用轻量化列表排版。
    - 全自动智能排序：系统底层严格按照“优先级 + 截止时间”进行自动排序，免除手动调整的心智负担。
    - 核心聚焦干预：不支持随意拖拽，仅支持将关键任务“置顶”以获取最高视觉权重。
    - 左侧色彩标识带，快速区分任务优先级或特定分类。
2. 节点化进度控制
    - 针对多阶段任务，采用“多区间/多节点”模型（如 1/3 节点）替代传统的父子任务嵌套。
3. 丰富的任务属性
    - 优先级设定（低、普通、高、紧急）。
    - 循环/周期性任务支持。
    - 灵活的完成条件设置（支持手动标记完成，或设定“时长到达”自动完成）。

三、 深度专注与计时系统 (计时页)
1. 多任务聚合计时
    - 支持在列表中同时勾选多个任务，进行合并计时。
2. 沉浸式单任务专注面板
    - 实时显示当前累计耗时与目标时长（如 300分钟）。
    - 可视化进度条（百分比显示）。
3. 快捷状态流转
    - 专注面板内可直接执行“标记完成”、“重置计时”、“归档”等操作，维持心流不断。

四、 系统级通知与快捷交互 (全局交互机制)
1. 贯穿生命周期的消息提醒
    - 精准推送任务的“开始提醒”与“结束提醒”，确保重要节点不被遗漏。
2. 互动式计时结束弹窗
    - 计时任务达成目标后，系统触发强感知的结束通知弹窗。
    - 下拉展开操作：通知弹窗支持下拉展开，直接提供“一键归档”快捷按钮，无需进入 App 即可顺滑完成任务生命周期闭环。
3. 应用内无缝直达
    - 点击消息通知主体，可直接唤起并精准跳转至 App 内部对应页面。

五、 碎片化灵感收集 (随手记/收集箱)
1. 灵活整理与打标
    - 支持快速录入文本并使用 `#标签` 语法进行分类。
    - 高自由度排版：支持随意拖拽调整条目位置，符合碎片化灵感整理的直觉。
2. 无缝晋升机制
    - 长按随手记内容，可直接将其“添加到任务”，转化为正式的执行待办。

六、 数据复盘与生命周期管理 (历史/归档页)
1. 颗粒度极高的专注日志
    - 记录任务的每一次计时起止动作及对应分段时长（精确至 < 1 min）。
2. 生命周期追溯
    - 完整记录任务的创建、执行与归档时间戳，直观对比目标与实际耗时。
3. 灵活的数据清理机制 (自主调控)
    - 支持精准删除具体的单条历史归档任务。
    - 支持按“月份”一键清空整月数据，赋予用户对长期积攒数据的完全控制权，避免冗余。
4. 容错回溯
    - 详情页提供“恢复为待办”功能，支持将已归档任务重新打回主执行流。

七、 个性化与系统设置 (设置页)
1. 极客级排版控制
    - 80% - 150% 字体大小无级调节，自带正文、标签实时预览。
2. 强大的主题引擎
    - 内置清晨、暗夜、海洋、深空等丰富情境主题，支持完全自定义。
3. 通知文案与国际化
    - 允许高度自定义“开始提醒”与“结束提醒”的具体播报文案。
    - 支持简体中文与英文无缝切换。
