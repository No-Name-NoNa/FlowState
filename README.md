# 流序 FlowState

流序是一款面向 HarmonyOS 的轻量计划与专注应用。它把“收集、整理、执行、复盘”放进一条清晰的日常流程里：先把想法放下，再安排成可执行的时间块，专注完成后回看自己的节奏。

## 核心理念

计划不应该变成负担。流序更关注当下能开始的下一步，而不是复杂的项目层级。每个任务都可以被拆成清晰的执行段，配合专注计时和历史复盘，让一天的安排更容易被看见、被调整、被完成。

## 主要模块

### 收集

用于快速记录临时想法、待办和灵感。收集箱只负责承接，不打断当前节奏；之后可以再通过标签、搜索和整理动作，把内容转为正式任务。

### 任务

用于管理当天和近期要推进的事项。任务支持优先级、标签、时间安排、子任务和备注，列表会围绕执行顺序展示，帮助用户把注意力放在下一件最值得开始的事情上。

### 专注

用于进入单段或多任务专注。计时页面强调沉浸感，保留开始、暂停、完成、放弃等核心操作，让用户不需要在专注过程中反复切换页面。

### 历史

用于查看专注记录和归档任务。历史页展示今日数据、近一周趋势和月度归档，帮助用户观察自己的专注变化，而不是只看总时长。

### 设置

用于调整主题、字体、提醒偏好和专注节奏。设置项只保留会直接影响体验的内容，避免把应用变成复杂配置面板。

## 功能范围

- 全局检索与标签筛选，覆盖收集、任务和归档数据。
- 任务优先级、截止时间、子任务、备注、周期任务和置顶能力。
- 专注计时与任务进度联动，支持计时分段记录和完成后归档。
- 通知提醒、开始提醒和完成提醒，帮助用户把握关键节点。
- 历史复盘展示今日专注、本周专注、活跃天数、近 7 天趋势和月度归档。
- 归档任务支持查看详情、恢复为待办、单条删除和按月清理。
- 主题、字体、语言和通知偏好可在设置中调整。

## 使用流程

1. 在“收集”里快速记下想法。
2. 把需要执行的内容整理成任务。
3. 为任务设置时间、优先级或子任务。
4. 进入“专注”开始一段清晰的执行。
5. 完成后在“历史”中回看节奏和成果。

## 设计方向

流序采用冷静、清爽、低干扰的视觉语言。界面尽量保留留白、柔和层次和清晰按钮，让用户可以一眼理解页面用途，同时减少不必要的视觉噪音。

## 技术栈

- HarmonyOS
- ArkTS
- ArkUI
- 本地数据持久化与应用状态管理
